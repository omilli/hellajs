Programmatically navigates to a different route. It handles URL construction with dynamic parameters and query strings.

## API

```typescript
function navigate(
  pattern: string,
  params?: Record<string, string>,
  query?: Record<string, string>,
  options?: { replace?: boolean; hash?: boolean }
): void
```

## TypeScript

The function is fully typed for parameters and options.

```typescript
// Navigate to /users/42?sort=asc
navigate(
  '/users/:id', 
  { id: '42' }, 
  { sort: 'asc' }
);
```

## Basic Usage

Use `navigate` to trigger navigation from event handlers, effects, or anywhere in your application logic.

```typescript
import { navigate } from "@hellajs/router";

// Navigate to a static path
navigate("/dashboard");

// Substitute route parameters
// Navigates to /posts/hello-world
navigate("/posts/:slug", { slug: "hello-world" });

// Add query parameters
// Navigates to /search?q=reactive
navigate("/search", {}, { q: "reactive" });
```

## Navigation Options

- `replace: true`: Replaces the current entry in the browser's history stack instead of pushing a new one. This is useful for redirects, like after a login, to prevent the user from navigating "back" to the previous page.
- `hash: true | false`: Overrides the router's configured mode (hash or history) for a single navigation call.

```typescript
// Redirect after login, replacing the history entry
navigate("/dashboard", {}, {}, { replace: true });
```
