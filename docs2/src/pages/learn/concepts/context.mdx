**Note**: HellaJS doesn't have built-in context APIs like React. Use prop passing, stores, or global signals for sharing data across components.

## Data Sharing Patterns

### Using Props (Recommended)

Pass data through component props for most cases:

```jsx
import { signal } from '@hellajs/core';

const App = () => {
  const theme = signal('dark');
  const user = signal({ name: 'John', email: 'john@example.com' });
  
  return (
    <div>
      <Header theme={theme} user={user} />
      <MainContent theme={theme} user={user} />
    </div>
  );
};

const Header = ({ theme, user }) => {
  return (
    <header class={`theme-${theme()}`}>
      <h1>Welcome, {user().name}!</h1>
    </header>
  );
};
```

### Using Global Signals

For truly global state, create signals outside components:

```jsx
// globals.js
import { signal } from '@hellajs/core';

export const theme = signal('light');
export const user = signal(null);
export const settings = signal({ 
  language: 'en', 
  notifications: true 
});
```

```jsx
// App.jsx
import { theme, user, settings } from './globals';

const Header = () => {
  return (
    <header class={`theme-${theme()}`}>
      <h1>{user() ? `Welcome, ${user().name}!` : 'Please login'}</h1>
    </header>
  );
};
```

### Using Store Pattern

For complex shared state, use HellaJS store:

```jsx
import { store } from '@hellajs/store';

// Create shared store
const appStore = store({
  user: null,
  theme: 'light',
  settings: {
    language: 'en',
    notifications: true
  },
  
  // Actions
  login(userData) {
    this.user = userData;
  },
  
  setTheme(newTheme) {
    this.theme = newTheme;
  },
  
  updateSettings(newSettings) {
    this.settings = { ...this.settings, ...newSettings };
  }
});

// Use in components
const UserProfile = () => {
  if (!appStore.user) {
    return <div>Please log in</div>;
  }
  
  return (
    <div class={`theme-${appStore.theme}`}>
      <h1>Welcome, {appStore.user.name}!</h1>
      <p>Language: {appStore.settings.language}</p>
      <button onclick={() => appStore.setTheme('dark')}>
        Switch to Dark Mode
      </button>
    </div>
  );
};
```

## Best Practices

### When to Use Each Pattern

**Use Props when:**
- Data is used by only a few components
- Parent-child relationships are clear
- You want explicit data flow

**Use Global Signals when:**
- Data is truly global (theme, user session)
- You need simple reactive values
- Component tree depth is shallow

**Use Store when:**
- Complex state with multiple related values
- Need actions/methods to modify state
- Managing application-wide state

### Performance Considerations

```jsx
// ✅ Good: Separate concerns
const theme = signal('light');
const user = signal(null);
const notifications = signal([]);

// ❌ Avoid: One large global object  
const globalState = signal({
  theme: 'light',
  user: null, 
  notifications: [],
  // ... everything
});
```
