Build your first HellaJS app in **under 5 minutes**. You'll create a reactive counter that updates automatically when you click it.

## What You'll Build

A simple counter button that increments when clicked, demonstrating:
- Reactive state with signals
- Event handling
- Automatic UI updates

## Setup

Create a new Vite project:

```bash
npm create vite@latest counter-app -- --template vanilla
cd counter-app
npm install
npm install @hellajs/core @hellajs/dom
npm install --save-dev vite-plugin-hellajs
```

Configure Vite in `vite.config.js`:

```js
import { defineConfig } from 'vite';
import viteHellaJS from 'vite-plugin-hellajs';

export default defineConfig({
  plugins: [viteHellaJS()]
});
```

## The Counter Component

Replace `main.js` with your counter:

```jsx
import { signal } from "@hellajs/core";
import { mount } from "@hellajs/dom";

const Counter = () => {
  const count = signal(0);

  return (
    <button onclick={() => count(count() + 1)}>
      Clicked {count} times
    </button>
  );
};

mount(Counter, "#app");
```

Start the dev server:

```bash
npm run dev
```

**That's it!** Click the button and watch it update automatically.

<Aside type="note">
  **What just happened?** The `signal(0)` creates reactive state. When you call `count(count() + 1)`, HellaJS automatically updates only the button text in the DOM.
</Aside>

## Understanding the Code

Let's break down what makes this work:

### 1. Creating Reactive State
```jsx
const count = signal(0);
```
- `signal()` creates a reactive container for your data
- It starts with the value `0`
- When the value changes, connected UI updates automatically

### 2. Reading and Writing State
```jsx
count()        // Read the current value
count(5)       // Write a new value
count(count() + 1)  // Update based on current value
```

### 3. Reactive Binding in JSX
```jsx
<button onclick={() => count(count() + 1)}>
  Clicked {count} times
</button>
```
- `{count}` creates a reactive binding (don't call it with `()`)
- When `count` changes, only the text updates in DOM
- No virtual DOM, no re-renders, no reconciliation

## Enhance Your Counter

Add some features to make it more interesting:

```jsx
import { signal, computed } from "@hellajs/core";
import { mount } from "@hellajs/dom";

const EnhancedCounter = () => {
  const count = signal(0);
  const isEven = computed(() => count() % 2 === 0);
  const message = computed(() => 
    count() === 0 ? "Click to start!" : 
    isEven() ? "Even number üéØ" : "Odd number üé≤"
  );

  const increment = () => count(count() + 1);
  const decrement = () => count(Math.max(0, count() - 1));
  const reset = () => count(0);

  return (
    <div style="text-align: center; padding: 2rem;">
      <h1>Counter: {count}</h1>
      <p style={`color: ${isEven() ? 'green' : 'blue'}`}>
        {message}
      </p>
      <div style="gap: 1rem; display: flex; justify-content: center;">
        <button onclick={decrement}>-</button>
        <button onclick={increment}>+</button>
        <button onclick={reset}>Reset</button>
      </div>
    </div>
  );
};

mount(EnhancedCounter, "#app");
```

### New Concepts Introduced

**Computed Values:**
```jsx
const isEven = computed(() => count() % 2 === 0);
```
- `computed()` creates derived values that update automatically
- `isEven` recalculates whenever `count` changes
- More efficient than recalculating in render

**Dynamic Styling:**
```jsx
style={`color: ${isEven() ? 'green' : 'blue'}`}
```
- Reactive bindings work in any attribute
- Style updates automatically when `isEven` changes

## Common Mistakes

**‚ùå Calling signals in JSX bindings:**
```jsx
<div>{count()}</div>  // Static value, won't update
```

**‚úÖ Pass signal function for reactivity:**
```jsx
<div>{count}</div>    // Reactive, updates automatically
```

**‚ùå Reading computed values in definition:**
```jsx
const message = `Count is ${count()}`;  // Static string
```

**‚úÖ Use computed for dynamic derivations:**
```jsx
const message = computed(() => `Count is ${count()}`);  // Reactive
```
