# Migrating from SolidJS

If you're coming from SolidJS, HellaJS will feel very familiar. Both are based on fine-grained reactivity.

## Similarities

- `createSignal` -> `signal`
- `createMemo` -> `computed`
- `createEffect` -> `effect`
- No Virtual DOM.
- Components run once.

## Key Differences

| SolidJS Feature | HellaJS Approach | Notes |
|---|---|---|
| `<For>` component | `forEach()` helper | HellaJS uses a helper function inside JSX. |
| `<Show>` component | Ternary or `&&` | Use standard JavaScript expressions for conditional rendering. |
| `props.children` | `children` argument | Children are passed as the second argument to a component function. |
| Stores | `store()` package | HellaJS provides a separate `@hellajs/store` package for deeply reactive objects. |

## Component Example

**SolidJS:**
```jsx
import { createSignal, createMemo } from 'solid-js';

function Counter() {
  const [count, setCount] = createSignal(0);
  const isEven = createMemo(() => count() % 2 === 0);

  return (
    <div>
      <p>Count: {count()}</p>
      <p>{isEven() ? 'Even' : 'Odd'}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}
```

**HellaJS:**
```jsx
import { signal, computed } from '@hellajs/core';

function Counter() {
  const count = signal(0);
  const isEven = computed(() => count() % 2 === 0);

  return (
    <div>
      <p>Count: {count}</p>
      <p>{() => isEven() ? 'Even' : 'Odd'}</p>
      <button onclick={() => count(count() + 1)}>Increment</button>
    </div>
  );
}
```
The main difference is how reactive values are passed to JSX. HellaJS automatically handles unwrapping signals in JSX, so you pass the signal function itself.
