# Migrating from React

This guide is for developers with React experience who are learning HellaJS.

## Key Differences

| React Concept | HellaJS Equivalent | Notes |
|---|---|---|
| `useState` | `signal()` | HellaJS signals are functions. `count()` to read, `count(1)` to write. |
| `useEffect` | `effect()` | HellaJS effects automatically track dependencies. No dependency array needed. |
| `useMemo` | `computed()` | HellaJS computed values are also dependency-tracked automatically. |
| Virtual DOM | None | HellaJS uses fine-grained reactivity to update the DOM directly. |
| Component Re-renders | None | Components run once. Only reactive parts of the JSX update. |

## Component Example

**React:**
```jsx
import { useState, useMemo } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  const isEven = useMemo(() => count % 2 === 0, [count]);

  return (
    <div>
      <p>Count: {count}</p>
      <p>{isEven ? 'Even' : 'Odd'}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}
```

**HellaJS:**
```jsx
import { signal, computed } from '@hellajs/core';

function Counter() {
  const count = signal(0);
  const isEven = computed(() => count() % 2 === 0);

  return (
    <div>
      <p>Count: {count}</p>
      <p>{() => isEven() ? 'Even' : 'Odd'}</p>
      <button onclick={() => count(count() + 1)}>Increment</button>
    </div>
  );
}
```
Notice in HellaJS:
- `{count}` is passed directly to the JSX.
- The conditional text is wrapped in a function `() => ...` to make it reactive.
