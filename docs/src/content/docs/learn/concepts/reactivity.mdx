HellaJS uses **fine-grained reactivity** to update your UI automatically and efficiently. Unlike virtual DOM frameworks that re-render entire components, HellaJS updates only the specific DOM nodes that need to change.

## The Magic of Automatic Updates

See reactivity in action with this simple example:

```jsx
import { signal, computed } from '@hellajs/core';
import { mount } from '@hellajs/dom';

const App = () => {
  const name = signal('World');
  const greeting = computed(() => `Hello, ${name()}!`);

  return (
    <div>
      <h1>{greeting}</h1>
      <input 
        value={name} 
        oninput={e => name(e.target.value)} 
      />
    </div>
  );
};

mount(App, "#app");
```

When you type in the input:
1. `name` signal updates with the new value
2. `greeting` computed automatically recalculates  
3. Only the `<h1>` text content updates in the DOM
4. No component re-renders, no virtual DOM diffing

<Aside type="note">
  **Key Insight**: HellaJS creates direct connections between your data and DOM elements. When data changes, the connected DOM nodes update immediately.
</Aside>

## The Three Reactive Primitives

HellaJS reactivity is built on three core primitives that work together:

<CardGrid>
  <Card title="ðŸ“Š Signals" icon="star">
    **[`signal()`](/packages/core/signal)** - Reactive state containers
    ```jsx
    const count = signal(0);
    count(5); // Updates value
    console.log(count()); // Reads value
    ```
  </Card>
  <Card title="ðŸ”„ Computed" icon="rocket">
    **[`computed()`](/packages/core/computed)** - Derived reactive values
    ```jsx
    const doubled = computed(() => count() * 2);
    // Updates automatically when count changes
    ```
  </Card>
  <Card title="âš¡ Effects" icon="lightning">
    **[`effect()`](/packages/core/effect)** - Side effects that run when dependencies change
    ```jsx
    effect(() => {
      console.log('Count:', count());
    });
    ```
  </Card>
</CardGrid>
