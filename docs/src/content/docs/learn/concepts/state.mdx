State management in HellaJS is built on reactive primitives. From simple local state with signals to complex application state with stores, HellaJS provides the tools you need.

## Local State with Signals

For component-local state, use signals:

```jsx
import { signal } from '@hellajs/core';

const Counter = () => {
  const count = signal(0);
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onclick={() => count(count() + 1)}>+</button>
      <button onclick={() => count(count() - 1)}>-</button>
    </div>
  );
};
```

## Shared State with Stores

For complex, nested reactive objects:

```jsx
import { store } from '@hellajs/store';

const appStore = store({
  user: {
    name: 'John',
    email: 'john@example.com'
  },
  theme: 'light',
  todos: [
    { id: 1, text: 'Learn HellaJS', done: false }
  ]
});

// All nested properties are reactive
const UserProfile = () => (
  <div>
    <h1>Welcome, {appStore.user.name}!</h1>
    <p>Theme: {appStore.theme}</p>
    <p>Todos: {appStore.todos.length}</p>
  </div>
);
```

## Global State Patterns

### Simple Global State
```jsx
// global-state.js
import { signal } from '@hellajs/core';

export const globalCount = signal(0);
export const globalUser = signal(null);

// Use in components
const Header = () => (
  <div>
    Count: {globalCount()} | User: {globalUser()?.name || 'Guest'}
  </div>
);
```

### Store-based Architecture
```jsx
// stores/app-store.js
import { store } from '@hellajs/store';

export const appStore = store({
  auth: {
    user: null,
    isLoggedIn: false
  },
  ui: {
    theme: 'light',
    sidebarOpen: false
  },
  data: {
    posts: [],
    loading: false,
    error: null
  }
});

// Actions
export const login = (user) => {
  appStore.auth.user = user;
  appStore.auth.isLoggedIn = true;
};

export const toggleTheme = () => {
  appStore.ui.theme = appStore.ui.theme === 'light' ? 'dark' : 'light';
};
```
