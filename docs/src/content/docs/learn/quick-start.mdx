Build your first HellaJS app in **under 5 minutes**. You'll create a reactive counter that updates automatically when you click it.

## 🚀 Quick Setup

The fastest way to try HellaJS:

```bash
npm create vite@latest my-hella-app -- --template vanilla
cd my-hella-app
npm install
npm install @hellajs/core @hellajs/dom
npm install --save-dev vite-plugin-hellajs
```

Configure Vite in `vite.config.js`:

```js
import { defineConfig } from 'vite';
import viteHellaJS from 'vite-plugin-hellajs';

export default defineConfig({
  plugins: [viteHellaJS()]
});
```

Replace `main.js` with your first reactive component:

```jsx
import { signal } from "@hellajs/core";
import { mount } from "@hellajs/dom";

const App = () => {
  const count = signal(0);

  return (
    <button onclick={() => count(count() + 1)}>
      Clicked {count} times
    </button>
  );
};

mount(App, "#app");
```

Start the dev server:

```bash
npm run dev
```

**That's it!** Click the button and watch it update automatically. No virtual DOM, no re-renders—just direct, efficient updates.

<Aside type="note">
  **What just happened?** The `signal(0)` creates reactive state. When you call `count(count() + 1)`, HellaJS automatically updates only the button text in the DOM.
</Aside>

## 🔧 TypeScript Setup

For TypeScript projects, use the TypeScript template and add types:

```bash
npm create vite@latest my-hella-app -- --template vanilla-ts
cd my-hella-app
npm install
npm install @hellajs/core @hellajs/dom
npm install --save-dev vite-plugin-hellajs
```

Update your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "jsx": "preserve",
    "types": ["@hellajs/dom"],
    "moduleResolution": "bundler",
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"]
  }
}
```

## 🏗️ Build Something Real

Let's extend the counter into a more useful component:

```jsx
import { signal, computed } from "@hellajs/core";
import { mount } from "@hellajs/dom";

const App = () => {
  const count = signal(0);
  const isEven = computed(() => count() % 2 === 0);
  const message = computed(() => 
    count() === 0 ? "Click to start!" : 
    isEven() ? "Even number 🎯" : "Odd number 🎲"
  );

  const increment = () => count(count() + 1);
  const decrement = () => count(Math.max(0, count() - 1));
  const reset = () => count(0);

  return (
    <div style="text-align: center; padding: 2rem;">
      <h1>Counter: {count}</h1>
      <p style={`color: ${isEven() ? 'green' : 'blue'}`}>
        {message}
      </p>
      <div style="gap: 1rem; display: flex; justify-content: center;">
        <button onclick={decrement}>-</button>
        <button onclick={increment}>+</button>
        <button onclick={reset}>Reset</button>
      </div>
    </div>
  );
};

mount(App, "#app");
```

<details>
<summary>🧠 **Understanding This Example**</summary>

- **`signal(0)`** creates reactive state that can change
- **`computed()`** creates derived values that update when dependencies change  
- **`isEven()`** automatically recalculates when `count()` changes
- **`{count}`** in JSX displays the reactive value—no manual DOM updates needed
- **Event handlers** update signals, triggering automatic UI updates

</details>

## 🐛 Troubleshooting

### Common Issues & Solutions

**❌ `Uncaught ReferenceError: React is not defined`**

This happens when JSX transforms expect React. Make sure you're using the HellaJS plugins:
- ✅ Vite: `vite-plugin-hellajs`
- ✅ Rollup: `rollup-plugin-hellajs` 
- ✅ Babel: `@hellajs/babel-preset`

**❌ `Cannot read properties of undefined (reading 'mount')`**

Check your imports:
```jsx
// ✅ Correct
import { mount } from "@hellajs/dom";

// ❌ Common mistake
import { mount } from "@hellajs/core";
```

**❌ Components not updating when signals change**

Make sure you're passing the signal function, not calling it:
```jsx
// ✅ Correct - passes the signal function
<div>{count}</div>

// ❌ Wrong - calls the signal once, gets a static value
<div>{count()}</div>
```

**❌ TypeScript errors with JSX**

Ensure your `tsconfig.json` has:
```json
{
  "compilerOptions": {
    "jsx": "preserve",
    "types": ["@hellajs/dom"]
  }
}
```

<details>
<summary>🔍 **Still having issues?**</summary>

1. **Check your file extensions**: Use `.jsx` or `.tsx` for JSX files
2. **Verify plugin order**: HellaJS plugins should come first in your build config
3. **Clear your build cache**: Delete `node_modules/.cache` and `dist` folder
4. **Check console errors**: Browser dev tools often show helpful error messages

</details>
