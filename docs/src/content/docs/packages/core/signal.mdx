title: signal
description: A reactive function that holds a value and notifies subscribers when it changes.
---

Holds a value and notifies subscribers when it changes.

```typescript
import { signal } from '@hellajs/core';

const count = signal(0);

count(1);
console.log(count()); // 1
```

## API

```typescript
function signal<T>(initialValue: T): Signal<T>

interface Signal<T> {
  (): T;            // Get current value
  (value: T): void; // Set new value
}
```

## Usage with TypeScript

TypeScript can infer signal types automatically, or you can provide explicit types for better control.

```typescript
// Inferred type: Signal<number>
const count = signal(0);

// Explicit type: Signal<string | null>
const name = signal<string | null>(null);

// Complex type: Signal<User>
interface User {
  id: string;
  name: string;
}

const user = signal<User>({
  id: '1',
  name: 'John'
});
```

## Performance Tips

Avoid unnecessary signal updates to improve performance.

```typescript
// ❌ Bad: Creates new object every time
const user = signal({ name: 'John', age: 30 });
const updateAge = () => user({ ...user(), age: user().age + 1 });

// ✅ Good: Only update when value actually changes
const age = signal(30);
const name = signal('John');
const updateAge = () => age(age() + 1);
```

## Common Patterns

Use signals for reactive state that triggers UI updates.

```typescript
// Form state management
const form = {
  email: signal(''),
  password: signal(''),
  isValid: computed(() => 
    form.email().includes('@') && form.password().length >= 8
  )
};

// Loading states
const isLoading = signal(false);
const fetchData = async () => {
  isLoading(true);
  try {
    const data = await api.getData();
    return data;
  } finally {
    isLoading(false);
  }
};
```
