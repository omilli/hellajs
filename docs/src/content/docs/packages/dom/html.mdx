Create DOM elements using template literals for runtime templating. The `html` function is a template literal tag that parses HTML strings and creates virtual DOM nodes.

```typescript
import { html } from '@hellajs/dom';
import { signal } from '@hellajs/core';

const name = signal('World');
const className = signal('greeting');

// Template literal syntax with interpolation
const greeting = html`<div class="${className}">Hello ${name}!</div>`;

// With event handlers
const count = signal(0);
const counter = html`
  <div>
    <p>Count: ${count}</p>
    <button onclick="${() => count(count() + 1)}">
      Increment
    </button>
  </div>
`;
```

## Features

### Dynamic Attributes and Content

Template literals allow you to interpolate signals and functions directly into HTML:

```typescript
import { html } from '@hellajs/dom';
import { signal } from '@hellajs/core';

const isActive = signal(true);
const message = signal('Hello');

const element = html`
  <div 
    class="${() => isActive() ? 'active' : 'inactive'}" 
    data-message="${message}"
  >
    ${message}
  </div>
`;
```

### Event Handling

Event handlers can be interpolated directly into the template:

```typescript
const clickHandler = () => console.log('Button clicked!');

const button = html`
  <button onclick="${clickHandler}" class="btn-primary">
    Click me
  </button>
`;
```

### Nested Elements

You can compose complex structures by nesting elements:

```typescript
const title = signal('Welcome');
const content = signal('This is the content');

const header = html`<h1>${title}</h1>`;
const paragraph = html`<p>${content}</p>`;

const page = html`
  <div class="container">
    ${header}
    ${paragraph}
  </div>
`;
```

## Performance Note

:::caution[Performance Consideration]
Template literals have a runtime parsing cost compared to JSX, which is compiled at build time. For performance-critical applications, consider using JSX with a HellaJS plugin instead.
:::
